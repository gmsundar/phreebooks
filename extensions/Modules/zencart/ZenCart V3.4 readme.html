Upgrading from a previous version of the ZenCart interface

In ZenCart

  1. From the ZenCartFiles folder, copy the admin folder contents into your admin folder to overwrite the old Phreedom files. This will not overwrite any core files.

  2. From the ZenCartFiles folder, copy the includes folder contents into your includes folder (NOT admin folder) to overwrite the old Phreedom files. This will not overwrite any core files.

  3. The file /admin/orders_mod_phreedom.php needs to be merged with the original admin/orders.php file. Replace the four sections (search for PhreeSoft) with the four sections from the new file. If you do not have any more modules installed that customized the orders.php file, the file orders_mod_phreedom.php can simply be renamed orders.php.

  4. In 1.5.0, under Admin → Tools → Install SQL Patches, run the script
    installPhreedom.sql, which is located in the /ZenCartFiles directory. This will register the module so it appears in the menu.

    NOTE: From version 3.3 to version 3.4, many of the files that were named phreebooks have been renamed to phreedom. These old files are no longer needed and can be removed. However, leaving them there will not cause a problem as the references to them have been removed.

In Phreedom

  1. Copy the files from the PhreedomFiles/modules folder to /modules folder. This will not overwrite any core files.

  2. If no other inventory customization is made, rename the two files in the /modules/inventory/custom/pages/main to extra_menus.php and extra_actions.php. If these files are already present, the two files must be merged together.

  3. Navigate to Company → Module Administration and Install the ZenCart module.

  4. Edit the ZenCart Properties and enter the information (this should be the same as your prior install). Note: If the module was present when Phreedom was upgraded, this step should have been already completed.

New Installation - Phreedom Side

  1. In php.ini ensure this option is set to On NOT Off: short_open_tag = On

  2. Unzip the ZenCart module files and copy the PhreedomFiles/modules directory into the phreedom installation modules directory. No files should be over-written.

  3. Navigate to modules/inventory/custom/pages/main and see if there were files previously present named extra_menus.php and extra_actions.php.

    If so, the contents of the files
    extra_menus_zencart.php
    extra_actions_zencart.php
    will need to be merged with these existing files and then can be deleted.

    If there are no extra_menus.php or extra_actions.php already present, simply rename them
    extra_menus_zencart.php => extra_menus.php
    extra_actions_zencart.php => extra_actions.php

Setup - Phreedom Side

For Order Downloads

  1. Navigate to Company → Module Administration and Install the ZenCart module.

  2. Edit the ZenCart Properties and enter the information.
    ZenCart path: full url to path where the admin folder is located (the additional pathing soap/.. will be appended to this)
    ZenCart user name/password: A admin user name and password to allow verification of the interface
    ZenCart tax class: Must match the textual description of your store. This will be matched to the Phreedom description to determine the taxes to add to the order.
    Price Sheets: Price sheet must exist in Phreedom for this feature to work.
    Status codes: Must match the numeric code of the status to set set ZenCart orders after downloading/confirming with Phreedom.

  3. Log in to Phreedom (you must have full rights to alter security settings of users) and select Company → Users.
    For each relevant user, go to the Tools tab and change the permissions for the ZenCart Interface to the desired access level.

  4. Create a new user (and password) and set all permissions to None except the ZenCart Interface (set to Full).
    This user name will be used by ZenCart to insert data into Phreedom.

  5. Log out and back in to reload the security settings.

  6. The install script will create a new tab in your inventory maintenance page along with three new fields. These new fields are required to properly sync your current products with the products in the ZenCart store. SETUP – ZenCart Side

  1. Go to Admin → Modules → Phreedom Interface
    If anything looks amiss or there is text missing above the install button, not all files have been uploaded correctly.
    Install/enable the order download capability.

  2. For the Phreedom XML URL enter: http://path_to_phreedom_root/soap/orders.php (or https:// if used)

  3. Enter the Phreedom database name. (used to select the proper company)

  4. Enter the ISO language (typically en_us) for error messages.

  5. Enter the Phreedom user name created for downloading orders.

  6. Enter the Phreedom password for the user.

  7. Enter an optional prefix (e.g. ZC) to use for creating the sales order number. Note Phreedom does not allow for duplicate Sales Order numbers. Adding a prefix will prevent conflict between downloaded orders and orders generated within Phreedom.

  8. If using multi_store, enter a store id here. This can be used for reporting activity through different sales channels. This field is not required and may be left blank.

  9. Enter a sales rep id. This field is not required and may be left blank.

  10. Enter the gl sales account number to post the order. This number must match to the chart of accounts in Phreedom or an error will result. Typically a sales type account is used.

  11. Enter the gl receivables account number. This number must match to the chart of accounts in Phreedom or an error will result. Typically a receivables type account is used.

  12. If you want the order status to be updated when an order is downloaded, select the status you want to set the order to.

  13. When importing an order Phreedom will auto-generate a customer id to create the customer account. Either the email address can be used or the billing telephone number (all non-numeric characters are stripped). This will then be used as the Customer ID within Phreedom.

  14. Categories in Phreedom must exist in Zen Cart and match identically. This is used to properly determine where to place the item in the Zen Cart catalog.

USAGE - Phreedom Side

Product Upload

  1. Inventory → Maintain
    Once the added fields have been entered into Phreedom, the catalog check box is tested for each inventory item and if selected, an upload icon will appear next to the inventory item in the action area of the main inventory screen. Click the button to upload the product.

  2. Tools → ZenCart Interface → Bulk Upload
    This action will upload all inventory items that have the catalog checkbox set in the inventory database.

  3. Tools → ZenCart Interface → Synchronize Products
    This feature will compare the selected inventory items from the Phreedom inventory database file and verify they match with the products listed on your Zen Cart e-store.
    Phreedom will not delete the items on the Zen Cart store but will identify items that are there but are not flagged to be there. They must be removed manually from Zen Cart.

  4. Tools → ZenCart Interface → Confirm Shipments
    This feature will change the status for all orders shipped on the selected date. Data is pulled from the shipping_log table and will include the carrier and tracking number.
    The order status will be changed to the value set during the ZenCart Administration setup.

Order Download

  1. Individual Order Download - Once an order has been placed, select Customers → Orders from the admin screen.
    An icon will appear in the action area next to each order to download the order (tooltip 'Download').
    Click the button to download the order to Phreedom. If successful, the page will reload and the icon will no longer appear. Repeat for all other orders.

  2. Bulk Download - The button at the top of the Orders page will attempt to download all orders that have the Download status flag (table orders, field phreebooks) set to zero. A status message will be displayed with the results of the bulk download attempt.

echo 'Submit to ' . ZENCART_URL . '/soap/' . $url . ' and XML string = <pre>' . htmlspecialchars($this->strXML) . '</pre><br />';


echo 'XML response (at the Phreedom side from Zencart) => <pre>' . htmlspecialchars($this->response) . '</pre><br />' . chr(10);

Error: Phreedom Zencart XML parse error: Processing Instruction not started at line 1

Check in php.ini that this option is On:

short_open_tag = On

echo 'category_id = ' . $result->fields['category_id'] . '<br>';

above this one

$this->strXML .= xmlEntry('ProductCategory', $result->fields['category_id']);
